Styles

In the new FCDO Design System, CSS styles are designed to be consistent, reusable, and scalable, ensuring that all government services provided by the department follow the same visual and functional guidelines.

Here's how the CSS styles work in the new FCDO design system:

Single CSS file

The FCDO Design System, shall use a single CSS file sits on top of the GOV.UK Design System styles to apply FCDO-specific branding, styling, and enhancements. This file ensures that the core GOV.UK styles remain intact, while introducing custom styles that reflect the FCDO's unique requirements.

Extends GOV.UK design system

The FCDO CSS file builds on the existing GOV.UK Frontend styles, leveraging its typography, grid system, and components.

Instead of modifying GOV.UK styles directly, the FCDO layer overrides and extends them, ensuring future compatibility when GOV.UK updates its design system.

Custom FCDO styles

The file introduces FCDO specific design SCSS variable (e.g., colors, spacing, typography).

It includes new components or alternative styling for elements that need to align with FCDO's branding.

For example, it may modify:

  • Primary color scheme (e.g., buttons, links, backgrounds)
  • Typography (e.g., heading font weights, text sizes)
  • Borders and shadows for enhanced visual differentiation
BEM (Block, Element, Modifier) naming

The CSS file follows the BEM methodology to keep styles modular and maintainable.

Classes follow the fcdo- prefix to differentiate from govuk- classes, for example:


        .fcdo-button {
        background-color: #005ea5; /* FCDO primary color */
        color: #ffffff;
        }
Overrides with minimal disruption

The file is lightweight and designed to override GOV.UK styles where necessary.

It ensures consistency across all FCDO services, without affecting the core GOV.UK framework.

Here is how a standard GOV.UK button might be modified for FCDO:


        /* default GOV.UK button */
        .govuk-button {
            background-color: #00703c;
        }
        
        /* FCDO override */
        .fcdo-button {
            background-color: #005ea5; /* FCDO blue */
            border-radius: 4px;
        }
Single source file

By maintaining one central CSS file, teams across FCDO can apply consistent styles without duplicating effort.

Updates to FCDO branding can be made globally by modifying this single file.
